# @mebelchi/app

The customer-facing app: **React + Vite**, wrapped with **Capacitor 7** for iOS
and Android. It recreates the 6-phase journey from `v7-journey.html` as real
screens, uses the `spike-3d.html` three.js setup for the live 3D constructor, and
drives the top price ticker from `priceProject` (`@mebelchi/pricing`).

## Run (web)

```sh
npm install
npm run dev        # Vite dev server
npm run typecheck  # tsc, app + node configs
npm run build      # production bundle → dist/
```

## Native (Capacitor)

```sh
npm run build
npm run cap:add:ios       # needs Xcode
npm run cap:add:android   # needs Android SDK
npm run cap:sync
```

`appId` / `appName` live in [capacitor.config.ts](capacitor.config.ts). The
`ios/` and `android/` projects are generated by `cap add` and are git-ignored.

## How it's wired

- **State** — one zustand store ([src/store.ts](src/store.ts)) holding the journey
  (`screen`, `qi`, quiz answers) and the editable run (`cabs`, materials, flags),
  plus `next()` / `back()` navigation over `FLOW`.
- **Pricing** — [src/model/toProject.ts](src/model/toProject.ts) maps the app's
  `Cabinet[]` to a `@mebelchi/schema` `Project`; [src/pricing/usePrice.ts](src/pricing/usePrice.ts)
  calls `priceProject(project, seedRateTable)` inside a store selector, so the
  ticker recomputes on every edit and shows nothing until there's a run to price.
- **Shared packages** — `@mebelchi/schema` and `@mebelchi/pricing` are aliased to
  their TypeScript source in [vite.config.ts](vite.config.ts) and
  [tsconfig.json](tsconfig.json) (no build step, no workspace install).

## Phases (built one at a time)

| Phase | Screen | Status |
| --- | --- | --- |
| А.1 | Quiz | **built** ([src/screens/QuizScreen.tsx](src/screens/QuizScreen.tsx)) |
| А.2 | Space / room | placeholder |
| Б | Variants | placeholder |
| В | Configure (3D constructor) | placeholder — three.js from `spike-3d.html` lands here |
| Г | Engineering | placeholder |
| Д | Cost | placeholder |
| Е | Handoff | placeholder |

Non-built phases render a routable placeholder so the flow and the price ticker
stay live while screens are filled in.
